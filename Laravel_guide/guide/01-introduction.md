# 01 - Introduction to Laravel 12 (2026 Edition)

Laravel is a PHP framework focused on developer productivity, expressive syntax, and a complete ecosystem for building modern backends. This guide focuses on API and backend development.

## Version Guidance (2026)

Laravel 12 is the current major release. Laravel follows a yearly release cadence with 18 months of bug fixes and 24 months of security fixes. Laravel 13 is planned for Q1 2026.

### Requirements (Laravel 12)

Laravel 12 requires PHP 8.2 or higher.

## Why Laravel for Backend

- **Ecosystem**: First-party tools for auth, queues, caching, and testing.
- **DX**: Artisan CLI, Eloquent ORM, and robust tooling.
- **Scalability**: Queues, events, Octane, and caching.
- **API-first**: Resources, validation, and middleware make APIs clean and consistent.

## What You Will Build

- REST APIs with validation and resources
- Token-based authentication with Sanctum
- Background jobs, events, and caching
- Production deployment patterns

## Common Backend Use Cases

- REST/JSON APIs
- Worker services and job processors
- Internal admin and tooling APIs
- Real-time features with WebSockets

## Guide Structure

1. Setup and tooling by OS
2. Core Laravel HTTP and database concepts
3. API development and authentication
4. Testing, queues, and events
5. Production and architecture patterns
